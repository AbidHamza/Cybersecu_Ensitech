# Exercice 1 : Sécuriser une API vulnérable

## Objectif

Identifier et corriger toutes les vulnérabilités dans l'API fournie.

## Instructions

1. Analysez le code de l'API vulnérable
2. Identifiez toutes les vulnérabilités (utilisez les connaissances des rooms précédentes)
3. Implémentez les corrections :
   - Validation d'input
   - Rate limiting
   - Authentification/Authorization
   - Gestion d'erreurs sécurisée
   - Headers de sécurité

## Checklist

- [ ] Validation complète des inputs
- [ ] Rate limiting implémenté
- [ ] Authentification sur endpoints sensibles
- [ ] Autorisation vérifiée
- [ ] Messages d'erreur génériques
- [ ] Headers de sécurité configurés
- [ ] Pas d'exposition de données sensibles

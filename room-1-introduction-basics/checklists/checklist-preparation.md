# Checklist de préparation technique

Avant de commencer les rooms pratiques (Room 2 et suivantes), vérifiez que votre environnement est prêt.

## Prérequis techniques

- [ ] Docker est installé et fonctionne (`docker --version`)
- [ ] Docker Compose est installé (`docker-compose --version`)
- [ ] Node.js version 18 ou supérieure est installé (`node --version`)
- [ ] Un éditeur de code est installé (VS Code, WebStorm, etc.)
- [ ] Git est installé (`git --version`)

## Vérifications Docker

- [ ] Docker Desktop est lancé (si sur Windows/Mac)
- [ ] Je peux lancer un conteneur de test : `docker run hello-world`
- [ ] Je comprends les commandes de base : `docker ps`, `docker-compose up`, `docker-compose down`

## Vérifications Node.js

- [ ] Node.js répond correctement
- [ ] npm est installé (`npm --version`)
- [ ] Je peux installer un package : `npm install` dans un projet de test

## Compréhension de base

- [ ] Je comprends ce qu'est un conteneur Docker
- [ ] Je comprends ce qu'est un fichier `docker-compose.yml`
- [ ] Je sais comment lire un fichier `package.json`
- [ ] Je comprends la structure de base d'une application web (frontend/backend)

## Prêt pour la pratique

Si toutes les cases sont cochées, vous êtes prêt pour les rooms pratiques !

Si certaines cases ne sont pas cochées, consultez la section "Préparer son environnement" du README principal ou la section "Dépannage" de cette room.

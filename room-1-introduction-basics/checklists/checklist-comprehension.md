# Checklist de compréhension - Room 1

Utilisez cette checklist pour vérifier que vous avez bien compris les concepts fondamentaux avant de passer à la Room 2.

## Concepts de base

- [ ] Je peux expliquer ce qu'est une vulnérabilité avec mes propres mots
- [ ] Je comprends la différence entre une vulnérabilité et une menace
- [ ] Je peux expliquer ce qu'est un risque et comment il se calcule
- [ ] Je comprends ce qu'est une attaque et comment elle exploite une vulnérabilité
- [ ] Je peux donner un exemple concret de chaque concept

## OWASP Top 10

- [ ] Je connais au moins 5 risques de l'OWASP Top 10
- [ ] Je peux expliquer brièvement ce qu'est une injection
- [ ] Je comprends ce qu'est l'authentification défaillante
- [ ] Je peux expliquer ce qu'est le XSS
- [ ] Je comprends pourquoi l'exposition de données sensibles est dangereuse

## Analyse de code

- [ ] Je peux identifier au moins 3 problèmes dans le code du scénario initial
- [ ] Je comprends pourquoi stocker des mots de passe en clair est dangereux
- [ ] Je peux expliquer pourquoi les messages d'erreur trop informatifs sont problématiques
- [ ] Je comprends l'importance de valider les entrées utilisateur
- [ ] Je peux expliquer pourquoi "ça fonctionne" ne signifie pas "c'est sécurisé"

## Mentalité de sécurité

- [ ] Je comprends que la sécurité concerne tous les développeurs, pas seulement les experts
- [ ] Je comprends pourquoi il faut penser sécurité dès le début du développement
- [ ] Je comprends que même les petites applications peuvent être ciblées
- [ ] Je sais que les frameworks ne rendent pas automatiquement le code sécurisé
- [ ] Je comprends l'importance de tester dans un environnement séparé

## Prêt pour la suite ?

Si vous avez coché la majorité des cases, vous êtes prêt pour la Room 2 !

Si certaines cases ne sont pas cochées, relisez les sections correspondantes du README ou consultez les scénarios dans `/scenarios/` pour approfondir votre compréhension.
